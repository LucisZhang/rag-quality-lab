# RAG Quality Lab

RAG Quality Lab 是一个本地评测工作台，用于比较检索流程、检查知识库变更，并为
RAG 实验建立可追溯的证据约定。

- 后续本地检查点 `6c887a1` 完成了 EnterpriseRAG-Bench S1 的 C2 适配器、
  确定性清单、后端感知验证器、运行器约定和测试，但这些提交尚未同步到公开仓库。
- 已验证的 C2 证据下限为 11,309 份 S1 文档、130 道 S1 可回答问题和 68 项通过的测试。
- C2 只验证数据与评测基础设施，不证明检索质量或答案质量。
- C3 检索 A/B 在执行前因时间盒到期而停止，没有生成指标，也没有替代或玩具流程结果。

仓库保留了旧版 Streamlit 工作流和 2026 年 4 月的实验文件，用于追溯来源。这些是
历史输出，不是当前 Portfolio v1 的结果：

  Pipeline B 为 `0.9438`，四舍五入后为 `0.809` -> `0.944`；
- 已保存的规模基准最多索引 50,000 份文档；
- 另一个由 MS MARCO 派生的语料在 2026-07-09 测得 498,725 条记录，并始终不进入 Git；
- 2026 年 7 月的验证只重新解析已提交文件并运行确定性检查，没有重新执行 judging，
  也没有产生新的答案质量证据。

这些数字只描述归档基线，不描述 EnterpriseRAG-Bench C2，也不得作为 C3 或当前模型
质量对比结果。

## 公开基线包含什么

- 用于查询检查、流程对比、回归检查和规模测试的 Streamlit 界面；
- `NaiveVectorRAG` 与 `HybridRerankRAG` 两种流程实现；
- 版本化的小型知识库与自编评测问题；
- 针对受控数据和已保存文件的确定性验证脚本；
- 不重新发布受限语料的合成结构样例。

这些源码能够展示实现方式，但不能单独证明当前模型质量。

## C3 无结果记录

C3 原计划在完整 S1 范围比较真实 dense 流程与真实 hybrid/rerank 流程。预检发现本地
环境缺少 Chroma、PyTorch、Transformers、sentence-transformers、LangChain 集成和
已缓存的 cross-encoder。安装完整依赖超出离线与磁盘时间盒，改用简单词法替代又会
使比较失效，因此运行在产生指标前停止。

## 基线快速开始

需要 Python 3.11、用于基线本地模型路径的 Ollama，以及足够的模型与索引磁盘空间。

```bash
python3.11 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements-lock-py311.txt
python scripts/verify_data.py
pytest
ruff check .
streamlit run app.py
```

模型下载、语料生成、judged evaluation 和大型索引都是显式本地操作，不属于 CI，也
不应只为复现 README 而执行。
